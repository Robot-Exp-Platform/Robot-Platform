# 02-3 状态收集器

## 相关关键词

> 多线程 状态收集器

## 任务目标

到目前为之，在 Exp 中 is_running 的实现还是非常可怜的

```rust
pub fn is_running(&self) -> bool {
    true
}
```

而实际上，这个判断函数应该是一个状态收集器，Exp 当然应该了解其下所有节点有多少节点在初始化，有多少节点已经开始 updating ，有多少节点在运行时超过了预设的时间周期，有多少节点任务运行完成闲着没事干。

所以，我们需要一个状态收集器，收集管理所有节点的状态，包括但不限于：

- 初始化中
- 运行中
- 运行超时
- 运行完成，等待任务
- 炸了

新建一个枚举类型名为 NodeStatus ，并在 Exp 中实现一个状态收集器，用于收集所有节点的状态。
